Overview

What Was Built

Two production REST APIs that connect JTL's BSS platform directly to network equipment, eliminating manual provisioning workflows and enabling automated, near-instant service delivery.

OLT Provisioning API
port 8000  ·  ZTE C300/C320/C600/C620/C650
Automates the full GPON ONU lifecycle — from a customer's ONT being plugged in to them receiving WiFi credentials on their phone — in a single BSS API call.
  • 14-step automated provisioning workflow over SSH
  • TR-069/ACS WiFi configuration via GenicACS
  • SMS, Email, and WhatsApp credential delivery
  • Suspend, activate, and package upgrade in real time
  • Persistent ONU inventory in MariaDB
M6K P2P Circuit API
port 8001  ·  ZTE M6000-S / M6000-8S
Provisions MPLS L2VPN pseudowire (VPWS) and local cross-connect circuits on the IP/MPLS backbone. One BSS API call creates a live P2P circuit across two PoPs.
  • VPWS pseudowire with auto-assigned VC ID (5000–9999)
  • Pushes xconnect config to both A-end and Z-end PEs
  • Polls PW state UP within 90 seconds
  • Suspend/activate via interface shutdown — config preserved
  • SQLite audit trail for every operation

How It Works

GPON Customer Activation Flow

What happens in the approximately 2 minutes between a BSS activation and a customer being online:

1
BSS sends one API call
POST /provision with customer ID, serial number, OLT name, and package ID.
2
ONU located on network
API scans GPON ports via SSH to find the ONU by its hardware serial number.
3
ONU authorized on OLT
API runs authorize + T-CONT + GEM port + service-port commands. ONU gets online.
4
ACS endpoint configured
TR-069 ACS URL pushed to ONU over CLI. ONU will phone home to GenicACS.
5
ONU phones home
API polls GenicACS for up to 120s until the ONU's TR-069 Inform is received.
6
WiFi configured remotely
Auto-generated SSID and password pushed via ACS TR-181 to both 2.4GHz and 5GHz bands.
7
Customer notified
WiFi credentials sent by SMS, Email, and WhatsApp simultaneously.
8
BSS receives confirmation
API returns ONU location, WiFi SSIDs, notification status, and total duration.
Zero touch after plug-in:

Once the customer's ONT is physically installed and connected to the PON port, the entire activation — OLT configuration, WiFi setup, and customer notification — is triggered by a single BSS action with no further manual steps.

VC ID management is automatic:

The API maintains a pool of MPLS VC IDs (5000–9999) and auto-assigns the next available one. No manual VC coordination between teams or risk of duplicate VC labels.

Security & Operations

Security Posture

Authentication
All API endpoints require X-API-Key header. Keys are configurable via environment variables. Invalid keys return 401 immediately.
Credential Encryption
OLT SSH credentials are encrypted at rest using Fernet symmetric encryption (AES-128-CBC). The key is stored separately from the database.
Audit Trail
Every provisioning, suspension, and deletion is logged with the API key used, commands sent to equipment, and outcome. Full traceability.
Internal Network Only
Both APIs run on the internal server at 172.16.37.18. Access from the public internet requires TLS termination via nginx reverse proxy.
