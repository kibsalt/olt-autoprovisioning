JTL OLT Provisioning API

REST API for automated ZTE OLT customer provisioning   Private Network

Base URL: http://172.16.37.18:8000/api/v1  |  Swagger UI: http://172.16.37.18:8000/docs

Overview

The OLT Provisioning API bridges JTL's BSS with the ZTE OLT fleet. It receives REST requests and translates them into SSH CLI commands on the OLTs.

OLT ModelPlatformInterface Syntax
ZTE C300ZXANgpon-olt_F/S/P
ZTE C320ZXANgpon-olt_F/S/P
ZTE C600TITANgpon_olt-F/S/P
ZTE C620TITANgpon_olt-F/S/P
ZTE C650TITANgpon_olt-F/S/P
The API server is on a private network (172.16.37.18). All OLT management IPs must be reachable from this server via SSH port 22.

Architecture

+---------+ REST/JSON +------------------------+ SSH/CLI +-------------+ | BSS | -----------------------> | OLT Provisioning API | --------------------> | ZTE C300 | | System | X-API-Key header | 172.16.37.18:8000 | Platform-specific | ZTE C320 | +---------+ | | ZXAN / TITAN | ZTE C600 | | Python 3.13 + FastAPI | commands | ZTE C620 | | systemd service | | ZTE C650 | +------+--------+--------+ +-------------+ | | +----+--+ +--+------+ |MariaDB| | SMTP | | 3306 | | Africa's| |(local)| | Talking | +-------+ +---------+

Authentication

All /api/v1/* endpoints require an API key in the X-API-Key header.

curl -H "X-API-Key: your-api-key" http://172.16.37.18:8000/api/v1/olts
ScenarioResponse
No API key provided401 — {"detail": "Not authenticated"}
Invalid API key401 — {"detail": "Invalid API key"}
Valid API key200 — Normal response

The /health endpoint does NOT require authentication.

Server Setup

1. Install Dependencies

sudo apt-get update
sudo apt-get install -y python3 python3-pip mariadb-server

sudo pip3 install --break-system-packages \
  fastapi uvicorn sqlalchemy aiomysql alembic asyncssh \
  pydantic-settings cryptography structlog africastalking \
  aiosmtplib jinja2 pymysql

2. Create Database

sudo mariadb -u root <<'SQL'
CREATE DATABASE IF NOT EXISTS olt_provisioning
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER IF NOT EXISTS 'olt_api'@'localhost' IDENTIFIED BY 'JtlOltDb2024!';
CREATE USER IF NOT EXISTS 'olt_api'@'%' IDENTIFIED BY 'JtlOltDb2024!';
GRANT ALL PRIVILEGES ON olt_provisioning.* TO 'olt_api'@'localhost';
GRANT ALL PRIVILEGES ON olt_provisioning.* TO 'olt_api'@'%';
FLUSH PRIVILEGES;
SQL

3. Generate Encryption Key

python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

4. Create .env File

See Configuration section for all variables.

5. Create Tables

cd /home/jtlacs/olt-provisioning-api
python3 -c "
from sqlalchemy import create_engine
from app.config import settings
from app.models.base import Base
import app.models
engine = create_engine(settings.sync_database_url)
Base.metadata.create_all(engine)
print('Tables created!')
engine.dispose()
"

6. Create systemd Service

sudo tee /etc/systemd/system/olt-api.service <<'EOF'
[Unit]
Description=JTL OLT Provisioning API
After=network.target mariadb.service

[Service]
Type=simple
User=jtlacs
WorkingDirectory=/home/jtlacs/olt-provisioning-api
EnvironmentFile=/home/jtlacs/olt-provisioning-api/.env
ExecStart=/usr/bin/python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable olt-api
sudo systemctl start olt-api

Security Hardening

The production server has these security measures applied:

UFW Firewall

sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 8000/tcp  # API
sudo ufw --force enable

fail2ban (SSH Protection)

# /etc/fail2ban/jail.local
[sshd]
enabled = true
maxretry = 5
bantime = 3600    # 1 hour ban
findtime = 600    # 10 minute window

SSH Hardening

PermitRootLogin no
MaxAuthTries 3
ClientAliveInterval 300
ClientAliveCountMax 2
X11Forwarding no

Kernel Hardening

SYN flood protection, IP spoofing protection, ICMP broadcast ignore, redirect blocking.

What happens during provisioning

1
SSH Finds next available ONU ID on the GPON port
2
SSH Authorizes ONU on the OLT (onu X type ZXHN_F680 sn ZTEGC8FA0001)
3
SSH Configures TR-069 ACS (URL: http://197.232.61.253:7547, user: ACS)
4
SSH Generates and configures WiFi (2.4GHz + 5GHz, WPA2-PSK)
5
SSH Applies service profiles (T-CONT, GEM port, service port, VLAN)
6
DB Saves ONU record with WiFi credentials to database
7
NOTIFY Sends WiFi credentials to customer via email + SMS

Full Provisioning Workflow

Complete end-to-end example for provisioning a new customer.

Prerequisites: OLT registered, VLANs created, bandwidth profiles created, service profiles created. These are one-time setup steps.
# Variables
API="http://172.16.37.18:8000/api/v1"
KEY="X-API-Key: bss-prod-key-change-me"

# =============================================
# ONE-TIME SETUP (do once per OLT/service tier)
# =============================================

# 1. Register OLT
curl -X POST "$API/olts" -H "$KEY" -H "Content-Type: application/json" \
  -d '{"name":"OLT-WESTLANDS-01","host":"10.10.1.1","model":"C600","ssh_username":"admin","ssh_password":"zte_pass"}'

# 2. Create VLAN
curl -X POST "$API/vlans" -H "$KEY" -H "Content-Type: application/json" \
  -d '{"vlan_tag":100,"name":"INTERNET","service_type":"internet"}'

# 3. Create bandwidth profiles
curl -X POST "$API/bandwidth-profiles" -H "$KEY" -H "Content-Type: application/json" \
  -d '{"name":"UP_10M","cir":10000,"pir":10240,"direction":"upstream","dba_type":"type3"}'

curl -X POST "$API/bandwidth-profiles" -H "$KEY" -H "Content-Type: application/json" \
  -d '{"name":"DOWN_50M","cir":50000,"pir":51200,"direction":"downstream","dba_type":"type3"}'

# 4. Create service profile
curl -X POST "$API/service-profiles" -H "$KEY" -H "Content-Type: application/json" \
  -d '{"name":"RESIDENTIAL_50M","service_type":"internet","upstream_profile_id":1,"downstream_profile_id":2,"vlan_id":1,"gem_port":1,"tcont_id":1}'

# =============================================
# PER-CUSTOMER PROVISIONING
# =============================================

# 5. Discover unregistered ONUs
curl "$API/olts/1/onus/unregistered?frame=1&slot=1&port=1" -H "$KEY"

# 6. Provision customer (this does everything automatically)
curl -X POST "$API/olts/1/onus" -H "$KEY" -H "Content-Type: application/json" \
  -d '{
    "serial_number":"ZTEGC8FA0001",
    "customer_id":"CUST-10042",
    "frame":1,"slot":1,"port":1,
    "onu_type":"ZXHN_F680",
    "service_profile_ids":[1],
    "customer_email":"john@example.com",
    "customer_phone":"+254712345678"
  }'

# 7. Verify status
curl "$API/olts/1/onus/1/status" -H "$KEY"

# =============================================
# LIFECYCLE MANAGEMENT
# =============================================

# Suspend (non-payment)
curl -X POST "$API/olts/1/onus/1/suspend" -H "$KEY"

# Re-enable (payment received)
curl -X POST "$API/olts/1/onus/1/enable" -H "$KEY"

# Remove (customer churn)
curl -X DELETE "$API/olts/1/onus/1" -H "$KEY"

After successful ONU provisioning, WiFi credentials are sent automatically if customer_email and/or customer_phone are provided.

OLT Driver Reference

The API auto-selects the correct driver based on OLT model.

ZXAN Platform (C300, C320)

# ONU Authorization
interface gpon-olt_1/1/1
  onu 1 type ZXHN_F680 sn ZTEGC8FA0001
exit

# WiFi Configuration
interface gpon-onu_1/1/1:1
  wifi ssid 1 name JTL_CUST001
  wifi ssid 1 auth-mode wpa2-psk
  wifi ssid 1 wpa-key MyPassword
  wifi ssid 1 enable true
  wifi ssid 5 name JTL_CUST001_5G
  wifi ssid 5 auth-mode wpa2-psk
  wifi ssid 5 wpa-key MyPassword
  wifi ssid 5 enable true
exit

# Service Port
service-port 10001 vport-mode gpon-onu_1/1/1:1 vlan 100 user-vlan 100 gemport 1

TITAN Platform (C600, C620, C650)

# ONU Authorization (note underscore/hyphen difference)
interface gpon_olt-1/1/1
  onu 1 type ZXHN_F680 sn ZTEGC8FA0001
exit

# WiFi Configuration
interface gpon_onu-1/1/1:1
  wifi ssid 1 name JTL_CUST001
  ...
exit

# Service Port (different syntax)
service-port 10001 gpon 1/1/1 onu 1 gemport 1 match vlan 100 action vlan 100
